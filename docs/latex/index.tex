\doxylink{namespace_vesper}{Vesper} is a lightweight 2D/\+3D engine and editor inspired by The\+Cherno\textquotesingle{}s Game Engine Architecture series for the Hazel engine. ~\newline
 It provides a renderer, scene \& entity system, editor UI, input handling, and utilities to build simple games and interactive applications in C++ (C++17).

It is generally usable as an easily modifiable base line for creating graphics and software applications.

\begin{quote}
The specific purpose is for the creation of Particle System demos, experiments, and visualizations.

{\itshape {\bfseries{It is not intended to be a full-\/featured game engine,}}}

But rather an application to create a range of visual effects and particle system simulations. \end{quote}

\hypertarget{index_autotoc_md4}{}\doxysection{\texorpdfstring{Build (Visual Studio 2022)}{Build (Visual Studio 2022)}}\label{index_autotoc_md4}

\begin{DoxyEnumerate}
\item Run the \textquotesingle{}Win-\/\+Gen\+Projects.\+bat\textquotesingle{} script from the repository root \textquotesingle{}Vesper/\+scripts/\+\+Win-\/\+Gen\+Projects.\+bat\textquotesingle{} to generate the Visual Studio solution file ({\ttfamily .sln}).
\begin{DoxyItemize}
\item This runs Premake5 which is located in \textquotesingle{}Vesper/\+\+Vendor/\+bin/\+premake\textquotesingle{}.
\end{DoxyItemize}
\item Open the solution in Visual Studio 2022.
\item Select the desired configuration (e.\+g. {\ttfamily Debug} or {\ttfamily Release}) and platform ({\ttfamily x64}).
\item Right-\/click the project you want to run ({\ttfamily Vesper-\/\+Editor} or {\ttfamily Sandbox}) and choose {\bfseries{Set as Startup Project}}.
\item Build and run (F5).
\end{DoxyEnumerate}

Troubleshooting:\+

\item Ensure the Windows SDK and Visual C++ toolset for VS2022 are installed.
\item If include or linker errors appear, confirm vendor project builds (GLFW, Glad, Im\+Gui, etc.) and that project dependencies are set correctly.
\item For unresolved externals, confirm platform (x86/\+x64) consistency across projects.
